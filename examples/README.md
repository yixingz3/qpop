# examples

Worked examples of the `qpop` discipline. The *engine* doesn't change between domains — a new domain is
a config, and for the bare ledger, not even that.

| Example | What it shows |
|---|---|
| [`ai_supply_chain/`](ai_supply_chain) | the full `SOURCE → GATE → EVALUATE` funnel on the worked AI-supply-chain thesis — many cards in, **0 admitted** that round (no action is the modal outcome) |
| [`template_domain/`](template_domain) | a copy-me starting point for a new bottleneck domain (rare earths, power grid, LNG, uranium, copper, defense industrial base, …) — fill in two configs, same engine |
| [`ml_benchmark/`](ml_benchmark) | **non-finance**: pre-register an ML eval claim *and its falsifiers* with the domain-agnostic ledger, *before* running the eval |

Finance is a deliberately adversarial testbed; [`ml_benchmark/`](ml_benchmark) shows the same discipline
outside it.
